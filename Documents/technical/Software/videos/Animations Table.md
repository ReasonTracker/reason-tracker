# Animations Table

This document is a companion to [Video Animation](./Video%20Animation.md) and [Animations Glossary](./Animations%20Glossary.md).
It maps animated items and move-together groups to the command families, planner operations, and animation rules that drive them.

The `Command` column lists the most direct authored trigger for the row, not every possible indirect cause.
The `Operation` column lists the planner operation or frame-driven driver that the shared video layer actually animates.
Rows are grouped by whether the row moves on its own or is defined by a move-together phase group.

## Moves Individually

| Item                     | Command                                                                                                                                 | Operation                                                                                            | Animation                                                                                                                                                                                                                              |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| score occurrence         | `claim/add`, `claim/update`, `claim/delete`, and any connect or disconnect command whose wave reaches that score occurrence             | `ScoreAdded`, `ClaimUpdated`, `ScoreUpdated`, `scaleOfSources`, `ScoreDeleted`                       | Enters with opacity and insert scale, interpolates position, size, confidence, relevance, and content against shared snapshots, and exits by fading and shrinking.                                                                     |
| confidence source span   | `claim/add` with confidence, `confidence/connect`, commands whose wave changes source-side confidence geometry, `confidence/disconnect` | `ConnectorAdded`, `ScoreAdded`, `ScoreUpdated`, `scaleOfSources`, `ConnectorDeleted`, `ScoreDeleted` | Reveals with pipe then fluid on structural add, updates route and width later, and can participate in the ordered `source and junction` whole-span phase when a junction exists.                                                       |
| confidence delivery span | `claim/add` with confidence, `confidence/connect`, commands whose wave changes target-side confidence geometry, `confidence/disconnect` | `ConnectorAdded`, `ScoreAdded`, `ScoreUpdated`, `scaleOfSources`, `ConnectorDeleted`, `ScoreDeleted` | Reveals, updates, or shrinks like a connector span; later `ScoreUpdated` junctioned updates can be held for a `junction to target` delivery phase, but `scaleOfSources` updates move it with the rest of the whole-graph scale change. |
| relevance span           | `claim/add` with relevance, `relevance/connect`, commands whose wave changes the targeted confidence connector, `relevance/disconnect`  | `ConnectorAdded`, `ScoreAdded`, `ScoreUpdated`, `scaleOfSources`, `ConnectorDeleted`, `ScoreDeleted` | Reveals as pipe then fluid on structural add, reroutes to the active junction visual during later updates, and shrinks on removal.                                                                                                     |
| connector junction       | `claim/add` with relevance, `relevance/connect`, commands whose wave reshapes a targeted confidence connector, `relevance/disconnect`   | `ConnectorAdded`, `ScoreAdded`, `ScoreUpdated`, `scaleOfSources`, `ConnectorDeleted`, `ScoreDeleted` | Reveals, reshapes, holds, or hides depending on phase; during a relevance add it can start as a temporary line junction and then resolve into the structural junction box.                                                             |
| graph bounds             | Any command that changes layout footprint                                                                                               | Any operation whose reduced snapshot changes layout bounds                                           | Interpolates graph width and height between snapshots so the rendered graph footprint follows layout changes.                                                                                                                          |
| camera move              | `CameraMove`                                                                                                                            | Frame-driven camera state, not an engine operation                                                   | Interpolates target screen position and zoom scale together, then solves translation from the active target point.                                                                                                                     |
| fade                     | `Fade`                                                                                                                                  | Frame-driven opacity state, not an engine operation                                                  | Interpolates opacity in and out over named Remotion sequences without changing graph state.                                                                                                                                            |

## Move Together Groups

| Item Group                                                  | Command                                                                                                                                               | Operation                                                                                                     | Animation                                                                                                                                                                                 |
| ----------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| structural membership envelope                              | `claim/add`, `claim/delete`, `confidence/connect`, `confidence/disconnect`, `relevance/connect`, `relevance/disconnect` when graph membership changes | `ClaimAdded`, `ConnectorAdded`, `ScoreAdded`, `incomingScoresChanged`, or the corresponding delete operations | Establishes a layout-safe graph membership state before later propagation, sort, and scale phases animate.                                                                                |
| confidence add: layout and claim group                      | `claim/add` with a confidence connection                                                                                                              | Structural add group through `incomingScoresChanged`                                                          | Existing score occurrences move to the new layout while the new score occurrence enters at its final layout position and the new connector visuals stay hidden.                           |
| confidence add: empty pipe group                            | `claim/add` with a confidence connection                                                                                                              | Structural add group through `incomingScoresChanged`                                                          | The new confidence connector pipe walls and pipe interior reveal from source to target with an open moving end while the fluid remains hidden.                                            |
| confidence add: confidence fluid group                      | `claim/add` with a confidence connection                                                                                                              | Structural add group through `incomingScoresChanged`                                                          | The new confidence fluid reveals through the already visible pipe and completes the structural add animation.                                                                             |
| relevance add: layout and claim group                       | `claim/add` with a relevance connection                                                                                                               | Structural add group through `incomingScoresChanged`                                                          | Existing score occurrences move to the reduced structural layout while the new relevance score occurrence enters and the new relevance connector and junction stay hidden.                |
| relevance add: relevance pipe and temporary junction group  | `claim/add` with a relevance connection                                                                                                               | Structural add group through `incomingScoresChanged`                                                          | The relevance pipe reveals to a temporary line junction that matches the current target confidence connector thickness, angle, and center.                                                |
| relevance add: relevance fluid group                        | `claim/add` with a relevance connection                                                                                                               | Structural add group through `incomingScoresChanged`                                                          | The relevance fluid reveals to the same temporary junction endpoint while the target confidence delivery span remains held.                                                               |
| relevance add: junction shape group                         | `claim/add` with a relevance connection                                                                                                               | Structural add group through `incomingScoresChanged`                                                          | The target confidence connector junction changes from the temporary line junction to the structural junction box while the relevance endpoint stays locked to the active junction visual. |
| relevance add: confidence delivery group                    | `claim/add` with a relevance connection                                                                                                               | Structural add group through `incomingScoresChanged`                                                          | The target confidence delivery span animates from the settled junction toward the target claim while the relevance endpoint remains locked to that same junction box.                     |
| existing junction update: source and junction group         | Any command whose later wave emits a `ScoreUpdated` on an already-junctioned confidence connector                                                     | `ScoreUpdated`                                                                                                | Shared layout, the confidence source span, the junction box, and any targeting relevance endpoint update together, while the delivery span stays held at its previous geometry.           |
| existing junction update: junction-to-target delivery group | Any command whose later wave emits a `ScoreUpdated` on an already-junctioned confidence connector                                                     | `ScoreUpdated`                                                                                                | The settled junction box stays fixed and the confidence delivery span catches up afterward using whole-span interpolation instead of a path-relative width sweep.                         |
